自我介绍 ​

你好，我叫王寒，毕业于东华理工大学软件工程专业，目前已经有6年的前端开发经验，技术上以 Vue 生态和 TypeScript 为主，对 Vue2、Vue3、工程化、性能优化、SSR、微前端都有比较深入的实践，也有 React、Node.js 和部分全栈开发经验。我的项目经历比较完整，既做过直接面向用户的业务系统，也做过偏平台型和基础设施型的项目。

在 BT 教育平台这个项目里，我负责过前端整体架构设计、技术管理、CI/CD、发布部署和性能优化，比较有代表性的成果是搭建官网 SSR 体系，把首屏加载时间从 2.1 秒优化到 0.5 秒；同时我也主导了公司级微前端架构升级，基于 qiankun 在 3 个月内整合了 10 多个不同技术栈的历史项目，解决了样式冲突、路由冲突、状态同步和部署耦合等问题。

另外我还从 0 到 1 设计过低代码平台，负责编排引擎、渲染引擎、变量系统和拖拽引擎这几块，重点解决复杂嵌套布局下的拖拽放置算法和复杂页面渲染性能问题。还有一个我比较想重点讲的是前端全链路监控平台，我主导设计了轻量级监控 SDK 和可视化分析平台，把前端错误、性能、行为轨迹、告警和 SourceMap 排障打通，覆盖了 10 多个核心业务，故障发现时效从用户反馈的小时级缩短到了系统告警的分钟级。

除了项目本身，我也比较关注研发效能和 AI 工具落地，平时会结合 Codex、Cursor、Claude Code 这些工具去优化需求拆解、Code Review、测试增强和工程交付效率。所以我现在更希望加入一个既重视业务结果、也重视技术质量和长期演进的团队，在前端业务开发之外，继续发挥我在架构、稳定性和协同推进方面的价值。”

项目经历 ​

BT教育平台 ​

项目描述：覆盖必提教育的 PC 端官网、管理后台、H5 端、Hybrid H5 页面、小程序等产品形态，包含课程体系、直播/录播听课系统、在线做题与自动判题、订单支付、营销中心、数据统计、教材、校友圈、社区问答等核心链路。

技术栈：Vue2/Vue3 全家桶、React 全家桶、TypeScript、UniApp、Vue Server Renderer、Vite、Webpack、qiankun、Egg.js 等。

个人职责： 1、负责前端项目整体架构设计、技术管理、业务开发、CI/CD 流程建设、项目优化、发布与部署等工作。 2、负责公司官网整体 SSR 架构设计与性能优化。 3、主导公司级微前端架构升级，基于 qiankun 搭建微前端基座。 4、负责公司研发效能提升与 AI 能力落地，搭建标准化 Code Review 流程并深度集成 CI/CD，推动存量 Vue2 项目向 Vue3 体系化升级，结合 Skills 构建测试增强方案，持续提升代码质量、测试覆盖率与工程交付效率。

项目难点： 1、官网首屏访问慢，SSR 性能较差。通过三层缓存架构、智能预加载、流式渲染、ISR 等手段提升服务性能，并通过服务降级、健康检查、监控告警、限流限频等机制保障服务高可用。 2、巨石应用难以维护和管理，不同技术栈维护成本高，升级与异构难度大，发布流程长且风险集中。基于 qiankun 搭建微前端基座，采用渐进式迁移方案，3 个月内将包含 Vue2/3、React16+、原生 JS 的 10+ 个历史项目完成应用集成，并解决技术栈混杂、重复开发、样式冲突、路由冲突、状态同步、应用通信、部署耦合等问题。 3、在线做题功能中，用户在弱网（地铁、电梯）、多端切换、意外关闭等场景下，经常出现答题记录丢失、数据不一致问题，高峰期日均投诉 30+ 条。为此设计并实现了一套基于事件溯源 + CRDT 的高可靠数据同步架构，包括答题数据 CRDT 结构、增量同步协议、轻量级冲突解决引擎、三层存储架构，以及配套的性能优化与监控体系。

项目成果： 1、性能指标显著提升：搭建 SSR 系统后，使首屏加载时间从 2.1s 降低到 0.5s，页面秒开率达到 90%，Core Web Vitals 评分达标，用户跳出率从 45% 降低到 28%。 2、采用渐进式迁移方案，3 个月内完成包含 Vue2/3、React16+、原生 JS 在内的 10+ 个历史项目集成；通过按需加载、公共依赖共享等方案，首屏加载性能提升 40%。同时开发微前端脚手架，一键生成子应用模板，接入成本从 3 天降至 2 小时，并建立微前端开发规范，输出技术文档，指导 5+ 开发人员接入。 3、设计并落地一套支撑百万级用户、日均十万次答题的高可靠数据同步系统，解决分布式环境下数据一致性、可用性和性能的平衡难题；数据丢失率从 5% 降至 0.03%，弱网答题成功率提升至 99.5%，用户投诉下降 95.8%。 4、实现 Code Review 自动化覆盖 30+ 前后端项目，测试覆盖率由 40% 提升至 80%，Vue3 升级覆盖核心业务 80% 模块，研发交付效率提升 30%，线上问题率下降 40%。

BT低代码平台 ​

项目描述：面向企业内部运营人员的自研低代码平台，旨在快速搭建复杂营销活动页和落地页，支持响应式与跨平台发布。平台包含可视化拖拽设计器、业务事件编排、数据模型驱动引擎、应用发布与版本管理全流程，支撑公司内部 300+ 营销活动专题页和落地页的搭建与运维。

技术栈：Vue3 全家桶、pnpm、TypeScript、Vite、interactjs、CodeMirror 6 等。

个人职责： 1、从 0 到 1 设计低代码平台整体架构，完成编排引擎、渲染引擎、变量系统、组件库、工程化和质量控制等核心能力建设；项目采用 Monorepo 微内核设计，并通过插件化机制扩展功能。 2、数据模型以及数据协议 Schema 设计、物料设计和物料管理平台搭建，以及动态 block 抽象设计。 3、负责编排引擎设计与性能优化，以及物料拖拽与编排交互设计；基于 mouse 事件自研拖拽放置引擎，实现组件在画布内的自由拖拽、缩放、辅助线对齐与吸附，并支持动态组件加载与实时布局调整。 4、变量编辑器系统架构搭建。基于 CodeMirror 的代码编辑能力，支持用户定义和管理全局变量与逻辑配置，并支持通过黑名单对象 + Proxy 创建沙箱运行环境，保证变量编辑器的安全性。 5、负责渲染引擎 Renderer 的开发与优化，通过策略模式进行数据转换与加工。

项目难点： 1、复杂嵌套布局下的拖拽放置算法较难处理。当容器嵌套多层（如卡片内套表格、表格内套输入框）时，精准判断组件拖拽目标容器与放置位置非常困难，容易出现错位或无法插入问题。通过采用射线检测法结合最近父容器回溯算法，不再仅依赖鼠标坐标下的元素，而是基于组件树中每个容器的层级与区域计算最合适的插入父节点，并将放置位置细分为“之前、之后、内部”三种状态，用蓝线高亮引导用户，放置准确率提升至 98% 以上。 2、性能瓶颈明显：在复杂活动页面（100+ 组件）下，拖拽组件或修改配置会触发全量渲染，配置反馈延迟高达 2-3 秒。通过分层优化策略，从响应式优化、渲染优化、拖拽优化、JSON 数据优化与增量更新等方面入手，将配置反馈延迟降至 200ms 以内，拖拽帧率稳定在 60 FPS。 3、低代码应用由配置生成，传统断点调试失效，问题定位困难。通过设计三层调试架构（业务层实时预览、实施层事件追踪、技术层性能火焰图，并通过权限系统进行控制），配合自定义调试面板，建立系统级/组件级追踪体系，覆盖事件流、性能、组件树、表达式执行、内存占用、用户操作、代码错误等场景，并通过表达式调试器、数据快照和数据对比等能力提升排障效率。

项目成果： 1、效率提升：平台上线 6 个月后，内部应用开发效率提升约 60%，原本需要 2 周开发周期的简单业务系统，现可在 2-3 天内完成搭建交付。 2、技术复用：封装 60+ 通用组件（容器、文本、图片、课程列表）与 30+ 业务组件（如优惠券、抽奖、倒计时等），覆盖投放引流、双十一、双十二、春节活动等营销投放场景，减少重复开发工作量。 3、性能优化：通过虚拟滚动、组件懒加载、缓存、防抖等方案，将复杂页面（50+ 组件）拖拽响应时间从 300ms 降至 50ms 以内，页面加载速度提升 40%。 4、业务覆盖：累计生成 200+ 线上可用页面，服务 10+ 核心项目部门，并通过完善的日志、追踪和性能分析工具链，让问题定位时间缩短 70%，线上 Bug 率降低 60%。 5、扩展能力：变量系统提供的「低代码片段」插槽机制，覆盖了 90% 以上的定制化需求，平台二次开发人力成本降低 70%。

前端全链路监控平台 ​

项目描述：为了解决线上问题“黑盒”现象及用户报障难以复现的问题，主导设计并落地公司内部前端监控 SDK 及可视化分析平台。平台实现了对 JavaScript 错误、资源加载失败、API 请求异常、白屏及用户核心行为轨迹的统一采集、分析与告警，覆盖公司主站及后台管理系统 10+ 个核心业务，日均处理百万级日志数据。

技术栈：Vue3、TypeScript、Vite、Next.js、AntV、Kafka、ClickHouse 等。

个人职责： 1、核心 SDK 开发：从 0 到 1 设计轻量级监控 SDK（压缩后 <30KB），采用插件化架构，支持错误、性能、行为埋点等能力按需启用，并通过统一上下文、事件总线和批量上报机制降低对业务代码的侵入。 2、性能监控：基于 Performance API 和 Web Vitals 采集 FCP、LCP、INP、CLS 等核心指标，建立页面性能基线；结合业务场景补充秒开率、业务关键区域渲染时间、卡顿等指标，支撑体验治理。 3、异常监控：设计统一异常上报体系，覆盖前端 JavaScript 运行时错误、资源加载失败、API 请求异常、自定义业务异常等场景，并结合 SourceMap 实现错误源码定位。 4、数据可视化与分析：基于 ClickHouse 搭建海量日志存储与查询层，结合 Vue3 + AntV G2 设计总览、错误分析、性能分析、用户行为路径等核心页面，帮助研发从“发现问题”快速走到“定位根因”。

项目难点： 1、首屏加载时间难以量化，广义白屏难以精准采集。通过设计基于 MutationObserver 的首屏检测算法，结合骨架屏过滤、关键资源加载与页面稳定窗口判定，将首屏时间检测准确率提升至 95% 以上，并能识别接口、数据加载、图片/视频资源导致的白屏问题。 2、性能预警容易演变为告警风暴，导致系统处理延迟和“告警疲劳”。通过引入告警抑制与降噪机制，实施告警分级与分渠道处理、冷热数据分离、限流与熔断等方案，保障监控系统在高噪声场景下的可用性与告警有效性。 3、海量日志下的采样与计算成本较高。针对性能数据在前端 SDK 侧设计动态采样策略，并在服务端进行实时聚合计算，生成 Web Vitals 趋势与分层分析结果，在保证统计代表性的同时控制计算与存储成本。

项目成果： 1、覆盖 95% 以上核心页面，解决过去线上问题“黑盒”状态，故障发现时效从用户反馈的小时级缩短至系统告警的分钟级。 2、基于监控平台对做题流程相关接口异常、超时分布和用户行为路径的关联分析，定位到网络抖动场景下关键链路稳定性问题；推动前后端协同优化超时策略、可恢复错误重试、提交态保护、接口幂等与失败兜底机制后，相关场景下做题流程异常率从 5% 降至 0.3%。 3、结合 SourceMap 实现错误源码定位，将线上 JS 异常排查从“依赖用户复现”缩短到“直接定位源码文件与代码行”，前端团队 Bug 排查效率提升 60%。 4、建立页面体验评级模型，基于监控平台发现课程详情页 LCP 高达 3.8s，并在可视化平台拆解资源加载与渲染耗时；通过懒加载、接口优化、渲染策略优化和图片方案升级等手段，使 LCP 中位数从 2.8s 降至 1.6s，并结合 A/B 测试持续监控“选课-下单-支付”核心漏斗页面，优化后订单转化率提升 3%。
