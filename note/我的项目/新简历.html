项目经历 ​

BT教育平台 ​

项目描述：包括必提教育的 PC 端官网、管理后台、H5 端、Hybrid H5 页面、小程序等产品，包括课程体系、直播/录播听课系统、在线做题与自动判题、订单支付、营销中心、数据统计、教材、校友圈、社区问答等核心链路。

技术栈：Vue2/Vue3全家桶、React全家桶、TypeScript、Uniapp、Vue-server-renderer、Vite、Webpack、qiankun、Egg.js等。

个人职责： 1、负责前端项目整体架构设计、技术管理和开发、CI/CD 流程、项目优化、发布和部署等工作。 2、负责公司官网整体 SSR 架构设计与性能优化。 3、主导公司级微前端架构升级，基于 qiankun 搭建微前端基座。 4、负责公司研发效能提升与 AI 能力落地，搭建标准化 Code Review 流程并深度集成 CI/CD，推动存量 Vue2 项目向 Vue3 体系化升级，结合 Skills 构建测试增强方案，持续提升代码质量、测试覆盖率与工程交付效率。

项目难点： 1、官网首屏访问慢，SSR 性能差，通过三层缓存架构、智能预加载、流式渲染、ISR等手段保证服务性能，并通过服务降级、健康检查、监控告警、限流限频等机制，保证服务的高可用。 2、巨石应用难以维护和管理，不同技术栈维护成本高，项目技术栈难以升级与异构，发布流程长且风险集中。基于 qiankun 搭建微前端基座，采用渐进式迁移方案，3个月内将包含 Vue2/3、React16+、原生JS 的10+个历史项目进行应用集成，并解决技术栈混杂、重复开发、样式冲突、路由冲突、状态同步、应用通信、部署耦合等问题。 3、在线做题功能中，用户在弱网（地铁、电梯）、多端切换、意外关闭等场景下，经常出现答题记录丢失、数据不一致的问题，高峰期日均投诉30+条。于是我设计并实现了一套基于事件溯源+CRDT 的高可靠数据同步架构，包括答题数据的CRDT 数据结构、增量同步协议、轻量级冲突解决引擎、三层存储架构，以及性能优化与监控体系，保证了用户的做题体验。

项目成果： 1、性能指标显著提升：搭建 SSR 系统后，使首屏加载时间从 2.1s 降低到 0.5s，页面秒开率达到 90%，Core Web Vitals评分达标，用户跳出率从 45% 降低到 28%。 2、采用渐进式迁移方案，3个月内将包含 Vue2/3、React16+、原生 JS 的 10+ 个历史项目进行应用集成，通过按需加载、公共依赖共享等方案，首屏加载性能提升40%。开发微前端脚手架，一键生成子应用模板，接入成本从3天降至2小时，并建立微前端开发规范，输出技术文档，指导 5+ 开发人员接入。 3、设计并落地了一套支撑百万级用户、日均十万次答题的高可靠数据同步系统，解决了分布式环境下数据一致性、可用性和性能的平衡难题。数据丢失率从 5% 降至 0.03%，弱网答题成功率提升至99.5%，用户投诉下降95.8%。 4、实现 Code Review 自动化覆盖 30+ 前后端项目，测试覆盖率由 40% 提升至 80%，Vue3 升级覆盖核心业务 80% 模块，研发交付效率提升 30%，线上问题率下降 40%。

BT一站式低代码平台 ​

项目描述：面向企业内部运营人员的一个自研低代码平台，旨在快速搭建复杂的营销活动页和落地页，将开发效率提升 50% 以上。并支持响应式和跨平台。平台包含可视化拖拽设计器、业务事件编排、数据模型驱动引擎、应用发布与版本管理全流程。支撑了公司内部 300+ 营销活动专题页和落地页的搭建与运维。

技术栈：Vue3全家桶、pnpm、TypeScript、Vite、interactjs、CodeMirror 6等。

个人职责： 1、从0到1设计低代码平台架构，包括编排引擎、渲染引擎、变量系统、组件库、工程化和质量控制等全功能搭建，项目采用 Monorepo 微内核设计，并通过插件化的方式来扩展功能。 2、数据模型以及数据协议 Schema 设计、物料设计和物料管理平台搭建，以及动态 block 抽象设计。 3、编排引擎设计、优化编排引擎性能，物料拖拽与编排交互设计，基于 mouse 事件自研了拖拽放置引擎，实现了组件在画布内的自由拖拽、缩放、辅助线对齐与吸附功能。支持动态组件加载与实时布局调整，提升系统的响应速度和灵活性。 4、变量编辑器系统架构搭建。基于 CodeMirror 的代码编辑能力，支持用户定义和管理全局变量与逻辑配置，并支持通过黑名单对象 + Proxy 创建沙箱运行环境，保证变量编辑器的安全性。 5、渲染引擎 Renderer 的开发和优化,通过策略模式来进行数据转换加工。

项目难点： 1、复杂嵌套布局下的拖拽放置算法。当容器嵌套多层（如卡片内套表格，表格内套输入框）时，精准判断组件拖拽的目标容器和放置位置变得非常困难，经常出现错位或无法插入的问题。采用 射线检测法 结合 最近父容器回溯算法。不再仅依赖鼠标坐标下的元素，而是通过计算组件树中每个容器的层级与区域，找到最合适的插入父节点，并将放置位置细分为“之前、之后、内部”三种状态，用蓝线高亮引导用户，放置准确率提升至 98% 以上。 2、性能瓶颈：拖拽卡顿问题，大量组件同时渲染，大数据 json 性能问题，在搭建复杂活动页面（100+ 组件）时，拖拽组件或修改配置会触发全量渲染，配置反馈延迟高达 2-3 秒。采用分层优化策略，通过响应式优化、渲染优化、拖拽优化、json数据优化以及增量更新的方式，配置反馈延迟降至 200ms 以内，拖拽帧率稳定 60 FPS。 3、黑盒困境：低代码应用由配置生成，传统断点调试失效，难以定位和调试问题。通过设计三层调试架构（业务层实时预览、实施层事件追踪、技术层性能火焰图，并通过权限系统进行控制），通过自定义调试面板，建立系统级/组件级别的追踪系统。监控事件流、性能、组件树、表达式执行、内存占用、用户操作、代码错误等场景，并通过表达式调试器，数据快照和数据对比等功能，帮助开发人员快速定位和解决问题。

项目成果： 1、效率提升：平台上线 6 个月后，内部应用开发效率提升约 60%，原本需要 2 周开发周期的简单业务系统，现可在 2-3 天内完成搭建交付。 2、技术复用：封装 60 + 通用组件（容器、文本、图片、课程列表）与 30 + 业务组件（如优惠券、抽奖、倒计时等），覆盖投放引流、双十一、双十二、春节活动等营销投放场景，减少重复开发工作量。 3、性能优化：通过虚拟滚动、组件懒加载、缓存、防抖等方案，复杂页面（50 + 组件）拖拽响应时间从 300ms 降至 50ms 内，页面加载速度提升 40%。 4、业务覆盖：累计生成 200 + 线上可用页面，服务 10+ 核心项目部门，并通过完善的日志、追踪和性能分析工具链，让问题定位时间缩短70%，线上 bug 率降低 60%。 5、扩展能力：变量系统提供的「低代码片段」插槽机制，覆盖了 90% 以上的定制化需求，平台二次开发人力成本降低 70%。

前端全链路监控平台 ​

项目描述：为了解决线上问题“黑盒”现象及用户报障难以复现的问题，主导设计了公司内部的前端监控SDK及可视化平台。实现了对JavaScript错误、资源加载失败、API请求异常、白屏以及用户核心行为轨迹的采集与报警，覆盖了公司主站及后台管理系统共 10+ 个核心业务，日均处理百万级日志数据。

技术栈：Vue3、TypeScript、Vite、Next.js、AntV G2、Web Vitals、Kafka、Clickhouse、Flink等。

个人职责： 1、核心SDK设计开发：从0到1设计轻量级SDK（压缩后<30KB），采用插件化架构，支持错误、性能、行为埋点等插件的按需加载，避免对业务首屏造成明显负担。 2、性能监控：基于Performance API和Web Vitals，采集FCP、LCP、INP、CLS等核心指标，建立性能基线。并支持秒开率、FMP、卡顿等指标。 3、异常监控：设计异常上报体系，能够捕获前端JavaScript错误、资源加载失败、API请求异常等，还支持自定义错误上报，便于开发团队添加业务逻辑中可能遇到的特定错误类型。 4、数据可视化与分析：搭建多维度分析面板，支持错误趋势、页面性能对比、用户行为路径追踪和版本回溯，帮助研发快速定位问题根因。

项目难点： 1、首屏加载时间难以量化，广义上的白屏难以采集。通过设计基于MutationObserver的首屏检测指标，将首屏时间检测准确率提高至95%以上。通过设计首屏检测算法，用于识别由于后端接口、数据加载、图片和视频加载过程中导致的白屏问题。 2、在做性能监控预警时遇到某个问题，因为性能问题不是短时间能解决掉的，导致预警邮件越发越多，后台拖慢了预警的速度，只能展示2个小时前的数据了，不仅会影响监控系统的性能，还会导致接收邮件的人产生“告警疲劳”，最终忽视真正重要的预警。最终通过引入“告警抑制”与“降噪”机制，实施告警分级与分渠道处理、系统架构解耦、冷热数据分离、限流与熔断等手段，保障监控系统的高可用性。 3、海量日志下的采样与计算。对于性能数据等海量日志，在前端SDK端实现动态采样率（根据页面流量或用户等级调整采样率），并在后端引入Flink进行流式计算，实时聚合Web Vitals指标，生成趋势图。

项目成果： 1、覆盖了95%以上的核心页面，解决了过去线上问题“黑盒”状态，故障发现时效从用户反馈（小时级）缩短至系统告警（分钟级）。。 2、基于监控数据，精准识别到“做题提交答案”接口在弱网环境下失败率高达 15%。通过推动后端合并接口及前端请求重试机制，将做题通过率提升至99.5%，显著提升了学员完课体验。 3、结合SourceMap实现错误源码定位，将线上JS异常的排查时间从“需要用户复现步骤”缩短至“直接定位代码行”，前端团队Bug排查效率提升60%。 4、建立页面体验评级模型，通过监控平台发现课程详情页LCP高达3.8秒，在可视化平台分解资源加载和渲染耗时，并通过懒加载、接口优化、渲染策略优化、图片方案升级等方案，使LCP（最大内容绘制）中位数从2.8s降低至1.6s，并通过AB测试监控“选课-下单-支付”核心漏斗页面，优化后订单转化率提升3%。
