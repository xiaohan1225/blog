# 详细说说低代码平台核心模块对应方案的选择以及架构实现，包含布局引擎、渲染引擎、数据源、流程引擎等

## layout-engine

### Flex 布局
类似于我们传统写代码 flex 布局的方式。兼顾灵活和易用。

- figma outLayout

### Block 布局
从上到下布局，没有左右之分，灵活度比较低

- 简单的无代码平台

### Grid 布局
参照格子点来布局，自动吸附在格子上。

- 仪表盘
- 可视化大屏

### Canvas 布局
设计类软件居多，100% 灵活的布局方式。

## material-renderer

平台的特定语言设计 DSL（Domain Specific Language）。JSONschema，整个平台编排完之后，存储的肯定是一个 json。

为了性能考虑，拆分为数据结构 json 和内容数据 json。

blockTree：
```js
[
  {
    id: 'container-a',
    type: 'container',
    children: [
      {
        id: 'text-a',
        type: 'text',
      },
      {
        id: 'icon-a',
        type: 'icon',
      },
       {
        id: 'button-a',
        type: 'button',
      }
    ]
  }
]
```

blocks: map结构，方便取数据
```js
{
  'container-a': {
    id: 'container-a',
    title: '默认标题'
  }
}
```
