## skills 是什么

Skills 把「长期积累的经验、规则、最佳实践」封装成一套可复用、可组合、可触发的技能

你可以把Skills理解为：
- AI的「专业知识模块」
- AI的「长期记忆 + 行为约束」
- 甚至是某个技术栈的「官方用法说明书（AI 版）」

## skills 特点
优点：
- 渐进式加载：先读头部源信息 -> 读框架总览 -> 具体框架内容

缺点：
- 标准统一，位置割裂。不同 AI 工具目录各异，跨平台使用不便
- 文件臃肿，无法渐进加载。大一统文件阻碍流动，难以分批到达
- 索引缺失，命名不明导致低效查询。命名混乱无索引，被迫加载多文件试错。
- 多 skills 冲突：过度安装的隐患。经验不足导致功能重叠，引发决策困境，效果很差。
- 存在相似技能时，缺乏智能提示，导致重复或冲突（未智能）

## 使用效果
- 强模型 + 好 Skills：模型底子好，Skills 如虎添翼，效果拔群。
- 弱模型 + 好 Skills：模型能力差，Skills 难以发挥，“巧妇难为无米之炊”
## skills 文件夹
头部元信息：
- name: 技能名称
- description: 描述：何时用，提供能力

正文内容
- 技能流程
- 相关信息
- 文件说明

资源文件夹（辅助内容）
- examples(示例)
- templates(模版)
- scripts（脚本）
- reference.md（索引）

## 打造专属 Skill
- 现成 Skills 的烦恼：质量不一，不适用，找不到
- 打造我的专属 Skills，基于个人需求，快速构建

## SKILL.md 组成

### FrontMatter(YAML 元信息)
```
---
name: 技能名称
description: 描述：能干什么，何时用，比如当用户提到周报、weekly report、本周总结时使用
---
```

可选字段：
- allowed-tools: 允许使用哪些工具
- context: 需要加载的上下文文件
- user-invocable: 是否可被用户直接调用

### 执行流程（markdown 正文）
skill 核心内容，告诉 AI 具体怎么做：先问什么，再做什么，输出什么格式，有什么约束。

## 和 System Prompt 区别
- System Prompt 特点：全局、永久加载，比如在 CLAUDE.md 写的所有规则，每一轮 AI 都会看到
- Skill 是模块化的，按需加载。

## 和 MCP 区别
- MCP：能力接口
- Skill：知识和流程
