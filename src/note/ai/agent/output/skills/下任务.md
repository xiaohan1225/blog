• 可以把这四个 skill 当成一条文章生产流水线，而不是同时调用。

  推荐协作顺序是：

  技术写作 skill
    -> 文章初稿 / 结构 / Mermaid 图表

  文章审校 skill
    -> 去 AI 味 / 修逻辑 / 调语气 / 改节奏

  Markdown Mermaid 转图片 skill
    -> 把文章里的 Mermaid 图转成 PNG，适配 mdnice / 公众号

  公众号封面生成 skill
    -> 根据最终标题和主题生成封面图

  关键原则

  1. 先写，再审，再做发布适配

     不要一开始就转 Mermaid 图片。因为审校阶段可能会改结构、删图、改图。如果太早转图片，后面还要重做。
  2. Mermaid 保留到文章定稿前

     写作阶段可以大胆用 Mermaid：

     ```mermaid
     flowchart TD
     A --> B


     等文章内容稳定后，再用 Mermaid 转图片 skill 统一处理。

  3. 封面最后做

     封面依赖最终标题、文章定位、副标题。如果标题还没定，封面容易返工。
  4. 两个自定义 skill 要么安装，要么显式指定路径

     你现在的两个自定义 skill 只是放在：

     /Users/btclass/github/blog/src/note/ai/agent/output

     如果想让 Codex 自动识别，建议放到：

     ~/.codex/skills/

     比如：

     ~/.codex/skills/markdown-mermaid-to-images/SKILL.md
     ~/.codex/skills/wechat-cover-generation/SKILL.md

     否则使用时要明确说：
     “使用 /path/to/SKILL.md 这个 skill”。

  你可以这样下任务

  我要写一篇公众号技术文章，请按四阶段协作：

  1. 使用 technical-blog-writing skill，先帮我完成文章大纲和初稿。
     主题是：<主题>
     目标读者是：<读者>
     文章风格是：<风格>
     需要包含 Mermaid 图表。

  2. 初稿完成后，使用 huashu-proofreading skill 做三遍审校：
     - 第一遍检查事实、逻辑、结构
     - 第二遍降低 AI 味，增加真人表达
     - 第三遍优化手机阅读节奏和排版

  3. 文章定稿后，使用 markdown-mermaid-to-images skill，
     把 Markdown 里的 Mermaid 代码块转成 PNG 图片，并替换成图片引用。

  4. 最后使用 wechat-cover-generation skill，
     根据最终标题和文章主题生成公众号封面提示词，必要时直接生成封面图。

  输出结果：
  - 初稿 Markdown
  - 审校后 Markdown
  - Mermaid 图片资源
  - 公众号封面图或封面提示词

  更好的做法

  你可以再沉淀一个“总控 skill”，比如叫：

  wechat-article-production

  它不负责具体写作，而是专门编排这四个 skill：

  选题 -> 大纲 -> 初稿 -> 审校 -> Mermaid 转图 -> 封面 -> 发布前检查

  这样以后你只要说：

  使用 wechat-article-production，帮我写一篇关于 XXX 的公众号文章。

  它就知道什么时候调用哪一个 skill。
