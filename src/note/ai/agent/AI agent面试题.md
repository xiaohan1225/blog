## RAG 优化

传统 RAG 是固定流程：用户问题向量化 -> 相似度检索 → prompt 拼接 -> 生成回答

想要优化，最主流的方案就是升级为 Agentic RAG。

它通过 Agent 自主决策，实现动态优化：

1. 先对复杂问题进行意图理解与拆解，生成多个子问题；
2. 根据问题类型智能路由，选择向量库检索、互联网搜索、数据库查询或调用工具；
3. 检索后进行信息融合、重排、去重；
4. 进入反思环节，自我评估信息是否足够、是否存在冲突或幻觉；
5. 如果不满足，自动重新构造查询、再次检索，形成多轮迭代；
6. 最终确认信息充分后，再生成准确、可靠的回答。


整个过程不是固定流水线，而是 Agent 自主思考、自主纠错的智能闭环。
